# 24 回忆卷解析与考点加固

## 使用定位

本文件根据回忆卷整理。题干完整、答案可唯一判断的题给出答案；题干或选项缺失的题标为“无法唯一确定”，但仍给出对应知识点和解题方法。

复习时不要把本文件当作押题，而要用它检查自己是否会：

- 用英文术语判断概念题。
- 对自动机、LR、语义环境、IR、trace、liveness、寄存器分配做手算。
- 写出中间步骤，而不是只写最终答案。

## 判断题

| 题目 | 判断 | 解释 |
|---|---|---|
| `(a|c)*|(a|c)*b(a|c)*` 与 `(a|c)*(b|epsilon)(a|c)*` equivalent | True | 左式是“没有 `b` 或恰好一个 `b`”，右式中间可选 `b` 或 `epsilon`，也是至多一个 `b`。 |
| There exists a grammar that is LL(1) but not LR(1) | False | LL(1) 文法可用 1 个 lookahead 做确定性自顶向下分析，属于 LR(1) 能处理的范围。常用结论：LL(1) 是 LR(1) 的子集。 |
| Instruction Select is to find an appropriate machine instruction for each intermediate tree node | False | 指令选择是用 machine instruction patterns/tiles 覆盖 IR tree，不是一节点一指令；一个大 tile 可覆盖多个 IR 节点。 |
| Maximum munch always produce optimal tiling | False | Maximal Munch 是贪心算法，简单快速，但不保证全局最低成本；optimal tiling 需要动态规划等方法。 |
| If precedence and associativity declarations are not specified, YACC shifts by default to resolve a shift/reduce conflict | True | Yacc/Bison 对 shift/reduce conflict 默认 shift；reduce/reduce conflict 默认选先出现的产生式，但应尽量用优先级/结合性显式解决。 |
| Given 5 languages and 4 target machines with an IR, we should implement 20 transitions | False | 没有 IR 是 `5*4=20`；有共享 IR 后大约是 `5` 个 front ends + `4` 个 back ends，即 `9` 个转换。 |
| Global graph coloring directly in Multiple Inheritance leave empty slots in objects | 题干不够清楚，倾向 False | 多继承布局会带来字段 offset 冲突和空洞问题；graph coloring 是一种减少冲突/复用 slot 的方法，不应表述为“直接导致空槽”。若考试原文是 naive layout leaves empty slots，则应为 True。 |
| Reference counting can reclaim all the unreachable objects | False | Reference counting 不能回收循环垃圾，例如 `A->B` 且 `B->A`，roots 不再指向它们时计数仍非零。 |
| If function has call-by-reference parameter, it can return address of this parameter without dangling-reference problem | 通常 True，但依赖实际对象生命周期 | call-by-reference 参数本质是 caller 对象的地址/引用，返回 referent 的地址不会指向 callee 自己的栈帧。若 caller 对象之后也结束生命周期，仍可能 dangling。 |
| Hashing table with external chaining is mentioned as nondestructive, efficient data structure in functional style symbol table | False | 外部链哈希表通常是高效的 imperative/destructive 结构；functional style symbol table 强调持久化/非破坏更新。 |
| A node `d` dominates a node `n` if some path from start node `s0` to `n` goes through `d` | False | dominator 要求从入口到 `n` 的所有路径都经过 `d`，不是 some path。 |

## 单选题解析

### 1. 正则表达式 `a|b(ab)[ab]*`

由于连接优先级高于并，表达式等价于：

```text
a | (b a b [ab]*)
```

能匹配：

```text
a
bab
baba
babb
babaa
...
```

不能匹配：

```text
epsilon
b
ab
ba
aa
```

缺少选项，无法给唯一选项。考试中先加括号再判断字符串是否属于语言。

### 2. LR(1) item 同状态

文法：

```text
A -> ( A ) | a
```

题目 item：

```text
[A -> ( . A ), $]
```

点后是非终结符 `A`，closure 要加入 `A` 的产生式。新 lookahead：

```text
FIRST(")" $) = { ")" }
```

所以加入：

```text
[A -> . ( A ), )]
[A -> . a, )]
```

给出的选项中正确的是：

```text
[C] [A -> .(A), )]
```

`[A]` 和 `[B]` 的 lookahead `$` 错；`[D]` 是移进并归约后的完整项，不在当前 closure 状态里。

### 3. Follow Set

题干缺文法，无法计算。标准步骤：

```text
1. FOLLOW(start) 加 EOF。
2. 对 A -> alpha B beta，把 FIRST(beta)-{epsilon} 加入 FOLLOW(B)。
3. 若 beta 可空或为空，把 FOLLOW(A) 加入 FOLLOW(B)。
4. 迭代直到不变。
```

### 10. Loop Variant / Loop Invariant

回忆题：

```text
i <- 0
j <- i
Loop:
  a <- j + 1
  b <- a * c
  c <- ...
  i <- i + 1
  if ... goto Loop
```

若题目问“which variable is not loop-variant / is loop-invariant”，在已知片段中：

- `j` 在循环外定义，循环内不改。
- `a = j + 1` 每轮计算结果相同，因此 `a` 是 loop-invariant computation 的结果。
- `b = a * c` 取决于 `c`，若 `c` 在循环内改变，则不是 invariant。
- `c` 在循环内定义，通常 variant。

在给出的选项 `[A] a [B] b [C] c [D] d` 中，若问“不随迭代变化的变量”，倾向选 `[A] a`。但题干缺完整 `c <- ...` 和问题原文，不能保证唯一。

### 12. 关于 Freeze

逐项判断：

| 选项 | 判断 | 解释 |
|---|---|---|
| A. freeze 是在不能 simplify 也不能 coalesce 后进行 | True | 典型流程：Simplify/Coalesce 都不能推进时 Freeze。 |
| B. freeze 优先查看 degree 较小的 move-related 节点 | True | 常从低度数 move-related 节点入手，冻结相关 moves 后可进入 simplify。 |
| C. freeze 将两个原来 move related 的边变成 non move related | 表述不准，倾向 False | freeze 是冻结 move，把 move 从待合并集合移到 frozenMoves，不是改变 interference edge。 |
| D. freeze 后可以继续 simplify 和 coalesce | True | 冻结后节点可能不再 move-related，可进入 simplify；其他 move 仍可能 coalesce。 |

若考试问“错误的是”，优先选 C。

### 15. Copying Collection 优点

不属于 copying collection 优点的是：

```text
[C] save memory
```

原因：copying collection 通常需要 from-space/to-space 两块空间，空间开销较大。它的优点包括：

- 能回收循环垃圾，因为基于 reachability。
- 消除外部碎片，使对象紧凑。
- 分配快，通常 bump pointer。

### 哪种变量需要存在栈帧里

确定正确项：

```text
[B] address taken 的
```

取地址变量必须有稳定内存地址，不能只放寄存器。

补充：

- 被内层函数引用的变量也通常 escape，需要放 frame。
- “其值被传给其他函数”不一定 escape；如果只是按值传递，不需要放 frame。若按引用传递或地址传出，则 escape。

### IR Tree 执行后 `a` 的值

题目：

```text
TEMP fp = 10
SEQ(
  CJUMP(LT, TEMP(fp), CONST(5), L1, L3),
  LABEL(L1),
  MOVE(TEMP(a), 1),
  JUMP(L3),
  LABEL(L2),
  MOVE(TEMP(a), 2),
  LABEL(L3)
)
```

判断：

```text
10 < 5 为 false
```

因此 `CJUMP` 跳到 `L3`，会跳过 `LABEL L1` 和 `MOVE(TEMP(a),1)`，也不会到 `L2`。所以 `a` 没有被赋新值。

给出的 `[1,2,5,10]` 都不是严格正确答案；若考题必须单选，原题可能漏了初始 `a` 值或跳转目标。按 IR 控制流语义，答案应是 `a` unchanged/undefined。

### Liveness Analysis 在什么过程中使用

正确：

```text
[A] register allocation
```

liveness 用于构造 interference graph，决定哪些 temporaries 不能共用寄存器。

### Liveness 定义题

选项 C 回忆为：

```text
如果变量 a 在 node n 的 live-in 中，则所有 proc[n] 的 live-out 中有 a
```

若 `proc[n]` 指 predecessors，则 C 是 True：

```text
out[p] = union in[s] for s in succ[p]
```

只要 `n` 是 `p` 的 successor，`a in in[n]` 就会使 `a in out[p]`。

若原文其实是 successors 或 all paths，则不能这样判断。需要完整选项。

### 不符合 Left Value 的选项

正确：

```text
[A] BINOP(ADD, CONST(1), CONST(2))
```

`BINOP` 计算一个值，不能作为赋值目标。`TEMP x` 和 `MEM(...)` 可以作为 `MOVE` 左侧，因此是 l-value。

### 构造规范树的错误说法

错误：

```text
[D] Eliminate all CJUMPs
```

canonicalization 会消除 `ESEQ`、处理嵌套 `CALL`、线性化 `SEQ`，但不会消除所有 `CJUMP`。`CJUMP` 是控制流的基本语句。

### Stack Frame 最精准表述

给出的选项中最准确的是：

```text
[C] 用来存放函数的局部变量，参数和返回地址的一片区域
```

更完整说法：stack frame/activation record 是一次函数调用在运行时的记录，通常包含参数、返回地址、old FP、static/dynamic link、saved registers、局部变量和 outgoing arguments。

## 大题一：`a(a|b)*b(aa|bb)`

语言含义：

```text
以 a 开头，并且以 baa 或 bbb 结尾的 a/b 串
```

因为中间 `(a|b)*` 可匹配任意 a/b 串，最后必须接：

```text
b aa
或
b bb
```

### 1. NFA 画法

若不强制 Thompson，可以按阶段画：

```text
start --a--> q1
q1 --a/b loop--> q1        // 对应 (a|b)*
q1 --b--> q2
q2 --a--> q3 --a--> final1
q2 --b--> q4 --b--> final2
```

注意这个 NFA 允许在任意位置猜“这里是最后的 b”，因此 `q1 --b--> q2` 与 `q1 --b--> q1` 可以同时存在，这正是 NFA 的非确定性。

### 2. 子集构造思路

若没有 epsilon 边，`epsilon-closure(T)=T`。从 `{start}` 开始：

```text
D0 = {start}
on a -> {q1}
on b -> empty

D1 = {q1}
on a -> {q1}
on b -> {q1,q2}

D2 = {q1,q2}
on a -> {q1,q3}
on b -> {q1,q2,q4}
...
```

继续直到没有新集合。包含 `final1` 或 `final2` 的 DFA 状态为终态。

### 3. 最小化

对子集构造得到的 DFA：

```text
P0 = {终态集合, 非终态集合}
```

然后按 `a/b` 转移模式反复拆分。考试中重点不是状态编号完全一样，而是：

- 每个 DFA 状态必须是 NFA 状态集合。
- 终态判断必须看集合中是否含 NFA final。
- 最小化必须写每轮划分依据。

## 大题二：LR(1) 文法 `S -> CC, C -> aC | b`

文法：

```text
0. S' -> S $
1. S  -> C C
2. C  -> a C
3. C  -> b
```

把 `$` 看作 EOF。Canonical LR(1) 项集可写为：

```text
I0:
  [S' -> . S $, $]
  [S  -> . C C, $]
  [C  -> . a C, a]
  [C  -> . a C, b]
  [C  -> . b, a]
  [C  -> . b, b]

I1 = goto(I0,S):
  [S' -> S . $, $]

I2 = goto(I0,C):
  [S -> C . C, $]
  [C -> . a C, $]
  [C -> . b, $]

I3 = goto(I0,a):
  [C -> a . C, a]
  [C -> a . C, b]
  [C -> . a C, a]
  [C -> . a C, b]
  [C -> . b, a]
  [C -> . b, b]

I4 = goto(I0,b):
  [C -> b ., a]
  [C -> b ., b]

I5 = goto(I1,$):
  [S' -> S $ ., $]

I6 = goto(I2,C):
  [S -> C C ., $]

I7 = goto(I2,a):
  [C -> a . C, $]
  [C -> . a C, $]
  [C -> . b, $]

I8 = goto(I2,b):
  [C -> b ., $]

I9 = goto(I3,C):
  [C -> a C ., a]
  [C -> a C ., b]

I10 = goto(I7,C):
  [C -> a C ., $]
```

循环边：

```text
I3 --a--> I3
I3 --b--> I4
I7 --a--> I7
I7 --b--> I8
```

Parsing table 填法：

- `I0` on `a` shift `I3`，on `b` shift `I4`，GOTO `S/I1`，`C/I2`。
- `I1` on `$` shift `I5`。
- `I2` on `a` shift `I7`，on `b` shift `I8`，GOTO `C/I6`。
- `I3` on `a` shift `I3`，on `b` shift `I4`，GOTO `C/I9`。
- `I4` on `a,b` reduce `C->b`。
- `I5` accept。
- `I6` on `$` reduce `S->CC`。
- `I7` on `a` shift `I7`，on `b` shift `I8`，GOTO `C/I10`。
- `I8` on `$` reduce `C->b`。
- `I9` on `a,b` reduce `C->aC`。
- `I10` on `$` reduce `C->aC`。

## 大题三：Tiger 语义环境

代码：

```tiger
let
  type arr = array of int
  var x := 10
  function f(x:int):int =
    let
      var a: arr := arr[5] of x
      function g(): int = x + a[0]
    in
      g()
    end
in
  f(3)
end
```

### Global / outer let scope

基础环境已有：

```text
TEnv: int, string
VEnv: print, flush, getchar, ...
```

外层 `let` 新增：

```text
TEnv: arr -> array of int
VEnv: x -> int
VEnv: f -> function(int) -> int
```

### Entering parameter scope of function `f`

进入 `f` 函数体时新开作用域：

```text
VEnv: x -> int     // parameter x，shadow outer var x
TEnv: arr -> array of int  // 从外层可见
```

`f` 自身和外层 `x` 仍在外层环境中，但名字 `x` 查找会先找到参数 `x`。

### Inside let block within `f`

处理：

```text
var a: arr := arr[5] of x
function g(): int = x + a[0]
```

新增：

```text
VEnv: a -> arr
VEnv: g -> function() -> int
TEnv: 无新增类型
```

### `g` 内部的引用

```text
x + a[0]
```

- `x` 绑定到 `f` 的参数 `x:int`，不是外层 `var x := 10`。
- `a[0]` 中 `a` 绑定到 `f` 内部 let 的局部变量 `a:arr`。
- `a[0]` 类型是 `int`。

### Type-check 是否成功

成功。理由：

```text
arr[5] of x: size 5 是 int，init x 是 int，得到 arr
g(): x:int + a[0]:int，返回 int
f(x:int):int 的 body 是 g():int
外层 f(3): 3:int，调用返回 int
```

## 大题四：IR Tree

### `a[i+2]`

已知 `a` 在栈上相对 `fp` 的 offset 为 20 bytes。若 `a` 是数组指针变量，先从 frame 取出数组基地址：

```text
a_ptr = MEM(BINOP(PLUS, TEMP fp, CONST 20))
index = BINOP(PLUS, TEMP i, CONST 2)
addr  = BINOP(PLUS, a_ptr, BINOP(MUL, index, CONST W))
value = MEM(addr)
```

合成 Tree IR：

```text
MEM(
  BINOP(PLUS,
    MEM(BINOP(PLUS, TEMP fp, CONST 20)),
    BINOP(MUL,
      BINOP(PLUS, TEMP i, CONST 2),
      CONST W)))
```

若 `W` 表示 word 数而不是 byte 数，还要乘以 `wordSize`。

### 条件赋值

源逻辑：

```text
if h < 9 or h > 21:
  off_work := 1
else:
  off_work := 0
```

短路 OR 的 IR：

```text
CJUMP(LT, TEMP h, CONST 9, Ltrue, Lcheck2)
LABEL Lcheck2
CJUMP(GT, TEMP h, CONST 21, Ltrue, Lfalse)

LABEL Ltrue
MOVE(TEMP off_work, CONST 1)
JUMP(NAME Ldone)

LABEL Lfalse
MOVE(TEMP off_work, CONST 0)

LABEL Ldone
```

关键：`or` 不是先算两个布尔值再相加，而是第一个条件为真时直接跳到 true 分支。

## 大题五：Basic Blocks 与 Longest Trace

原代码需要给 fall-through 位置补 label：

```text
B0:
  LABEL L0
  c <- a+b
  CJUMP(d<2, L4, L0_false)

B1:
  LABEL L0_false
  c <- a*b
  JUMP L1

B2:
  LABEL L1
  a <- b%2
  JUMP L4

B3:
  LABEL L2
  c <- d*a
  CJUMP(c>0, L1, L2_false1)

B4:
  LABEL L2_false1
  n <- n-2
  CJUMP(n<0, L3, L2_false2)

B5:
  LABEL L2_false2
  a <- a+1
  JUMP L4

B6:
  LABEL L3
  b <- a+2
  JUMP L4

B7:
  LABEL L4
  d <- c%2
  n <- n+2
  JUMP done
```

一种 longest trace：

```text
L0 -> L0_false -> L1 -> L4
```

也可以给：

```text
L2 -> L2_false1 -> L2_false2 -> L4
```

只要每个 trace 内控制流可连续，并且最终所有 basic blocks 被 trace covering 覆盖即可。

## 大题六：Liveness 与 Interference

回忆卷没有给出完整代码和 live-out 提示，不能唯一计算。标准作答格式：

```text
1. 给每条 statement 编号。
2. 写 use/def。
3. 写 succ。
4. 从最后一条开始反向迭代：
   out[n] = union in[s] for s in succ[n]
   in[n] = use[n] union (out[n] - def[n])
5. 对每条定义 d，把 d 与 live-out[n] 中除自身外的变量连边。
6. move d<-s 特殊：通常不连 d-s，记录 coalescing candidate。
```

考试如果给了部分 live-out，可以用它校验自己迭代结果。

## 大题七：Spill 与图着色

题目缺原始冲突图和部分 live-out，无法唯一完成数值答案。以下是标准解题流程。

### 1. Spill Priority

常见公式：

```text
spill priority = use frequency / degree
```

若循环平均执行 `n` 次，循环内使用/定义的变量频率要乘 `n`；循环外只算 1 次。

步骤：

```text
1. 对每个 temporary 统计 uses + defs。
2. 循环内出现的乘以 n。
3. 查原始干涉图 degree。
4. priority 越低，越适合 spill。
```

### 2. Spill Rewrite

若选择 spill `v` 到 frame slot `slot_v`：

```text
每次 use v 前:
  v_use_i <- MEM[slot_v]
  用 v_use_i 替代 v

每次 def v 后:
  v_def_i <- ...
  MEM[slot_v] <- v_def_i
```

新增 temporary 的 live range 通常很短。

### 3. New Interference Graph

重写后重新做 liveness。不能在旧图上简单删掉 `v`，因为新增 load/store 和 temporaries 会改变 live ranges。

### 4. Coalescing

- 与 precolored node 合并：用 George criterion。
- 普通 temporary 合并：用 Briggs criterion。

作答时要写：
