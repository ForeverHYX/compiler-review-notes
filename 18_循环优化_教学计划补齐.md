- 数组越界检查消除。

## Dominator

在 CFG 中，节点 `d` 支配节点 `n`，如果从入口到 `n` 的每条路径都必须经过 `d`。

记作：

```text
d dominates n
```

入口节点支配所有节点。每个节点支配自己。

### Immediate Dominator

`idom(n)` 是严格支配 `n` 的节点中离 `n` 最近的那个。所有 `idom` 构成 dominator tree。

### Dominator 迭代算法

小图手算可以用集合迭代：

```text
Dom(entry) = {entry}
对其他节点 n: Dom(n) = 所有节点集合
重复直到不变:
  Dom(n) = {n} union intersection(Dom(p) for p in pred(n))
```

直觉：要支配 `n`，必须支配 `n` 的每个前驱路径；所以取所有前驱支配集合的交集，再加上 `n` 自己。

算出 `Dom(n)` 后，`idom(n)` 是 `Dom(n)-{n}` 里最靠近 `n` 的节点。做题时可以画支配树检查：父节点必须支配子节点。

## Back Edge 与 Natural Loop

如果边：

```text
n -> h
```

且 `h` 支配 `n`，这条边是 back edge，`h` 是循环头。

natural loop 包含：

- header `h`
- tail `n`
- 所有能到达 `n` 且不经过 `h` 的节点

直觉：back edge 把控制流带回 header，形成循环。

### Natural Loop 构造算法

已知 back edge `n -> h`：

```text
loop = {h, n}
worklist = [n]
while worklist not empty:
  m = pop(worklist)
  for each predecessor p of m:
    if p not in loop:
      add p to loop
      add p to worklist
```

注意不要沿着 `h` 的前驱继续跑到循环外；`h` 是循环头，已经在集合中。

如果一条回边的目标不支配源节点，它不是 natural loop 的标准 back edge。

## Preheader

循环优化常需要在循环前插入代码。preheader 是循环 header 前的一个专门基本块，所有从循环外进入循环的边先进入 preheader，再进入 header。

```text
outside -> preheader -> header -> loop body
```

## Loop-Invariant Code Motion

循环不变式是每次迭代结果都一样的计算。

```c
while (i < n) {
  x = a + b;
  y = x * i;
  i = i + 1;
}
```

如果 `a`、`b` 在循环中不变，`a+b` 可以移到循环外：

```c
x = a + b;
while (i < n) {
  y = x * i;
  i = i + 1;
}
```

安全外提需要考虑：

- 操作是否无副作用。
- 定义是否支配循环内所有使用。
- 移动后不会引入异常或改变执行次数相关语义。

手算 LICM 可以按三步：

```text
1. 找循环中所有表达式用到的变量
2. 如果表达式的所有操作数都在循环外定义，或由已知循环不变式定义，则它是 loop-invariant
3. 只有当移动安全时，才放到 preheader
```

安全性常见限制：

- 表达式可能除零、越界、空指针异常时，不能随便提前到原本可能不执行的位置。
- 表达式有函数调用或内存写副作用时，不能当普通算术表达式外提。
- 如果定义没有支配循环内所有使用，外提后可能改变某些路径上的值。

## Induction Variables

归纳变量随循环迭代按固定规律变化：

```c
i = i + 1
j = 4 * i + c
```

`i` 是 basic induction variable；`j` 是 derived induction variable。

## Strength Reduction

把昂贵运算换成便宜运算。

例：

```c
for (i = 0; i < n; i++) {
  a[i] = b[4 * i];
}
```

`4*i` 每次乘法可以改成递增：

```c
t = 0
for (i = 0; i < n; i++) {
  a[i] = b[t]
  t = t + 4
}
```

## Induction Variable Elimination

如果某个归纳变量只用于计算另一个变量，而后者已经能直接维护，则可以删除冗余归纳变量。

## Bounds-Check Elimination

安全语言每次数组访问可能检查：

```text
0 <= i && i < length(a)
```

如果编译器能证明循环内所有 `i` 都在范围内，就可以把检查移出循环或删除。

关键依赖：

- 循环边界。
- 归纳变量范围。
- 支配关系。
- 数组长度是否循环不变。

## Loop Unrolling

循环展开减少分支开销，并暴露更多优化机会。

原始：

```c
for (i = 0; i < n; i++) {
  sum += a[i];
}
```

展开 2 次：

```c
for (i = 0; i + 1 < n; i += 2) {
  sum += a[i];
  sum += a[i+1];
}
for (; i < n; i++) {
  sum += a[i];
}
```

需要处理余数迭代。

## 常见误区

- 有回边不一定是自然循环，必须 header 支配 tail。
- 循环不变不代表一定能外提，异常和副作用可能阻止移动。
- 强度削弱常引入新变量，不是简单文本替换。
- 展开循环会增加代码大小，不一定总是更快。
- 删除 bounds check 必须保证所有路径安全。

## 练习

1. 给 CFG，计算每个节点的 dominator 集合。
2. 根据 back edge 找 natural loop。
3. 找出一段循环中的 loop-invariant computation。
4. 对 `a[i*4]` 做 strength reduction。
5. 对一个数组访问循环判断能否消除 bounds check。
6. 把一个循环按因子 2 展开，并处理余数。
