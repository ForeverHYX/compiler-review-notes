# 16 垃圾回收

## 本章解决什么问题

垃圾回收自动释放程序不再使用的堆对象。它属于运行时系统，但编译器必须提供足够信息，例如哪些寄存器和栈槽里保存的是指针。

PPT 重点：mark-and-sweep、reference counting、copying collection、compiler interface。  
弱化：generational、incremental 等书上有，但课件提示可能不作为考试重点。

## 本章考试能力清单

- 概念题：能解释 garbage、root set、reachability、live object、pointer map、type descriptor、safe point。
- 手算题：能模拟 mark-and-sweep 堆图、reference counting 计数表、Cheney copying 的 scan/next。
- 辨析题：能说明 reference counting 为什么不能回收循环垃圾，copying GC 为什么能消除碎片但不省内存。
- 英文题：看到 `from-space/to-space`、`forwarding pointer`、`remembered set`、`tricolor abstraction` 能说出用途。

## 什么是垃圾

直觉上，垃圾是“以后不会再用的对象”。但准确判断“以后不会再用”不可判定，因为这会涉及程序是否终止等问题。

实际 GC 使用保守近似：从 root set 出发不可达的堆对象一定可以回收。

```text
roots: registers, stack slots, globals
          |
          v
reachable heap objects = live
unreachable objects    = garbage
```

## Root Set

Root set 包含程序可直接访问的指针来源：

- 全局变量中的指针。
- 当前栈帧和调用栈中的指针。
- 寄存器中的指针。

问题：运行时怎么知道一个栈槽或寄存器里是整数还是指针？这就需要编译器提供 pointer map 或 descriptor。

## Mark-and-Sweep

两阶段：

1. Mark：从 roots 出发遍历对象图，标记可达对象。
2. Sweep：扫描堆，未标记对象加入 freelist，已标记对象清除 mark bit。

```mermaid
flowchart LR
  R[Roots] --> A[Object A]
  A --> B[Object B]
  C[Object C unreachable]
```

`A`、`B` 被标记，`C` 被回收。

### 手算例题

堆图：

```text
roots: r1 -> A, r2 -> D
A -> B
B -> C
D -> B
E -> F
F -> E
G -> none
```

Mark 阶段：

| 起点 | 可达对象 |
|---|---|
| `r1 -> A` | `A,B,C` |
| `r2 -> D` | `D,B,C` |

最终 marked：

```text
{A,B,C,D}
```

Sweep 阶段扫描堆：

| 对象 | marked? | 处理 |
|---|---|---|
| A | 是 | 清 mark bit，保留 |
| B | 是 | 清 mark bit，保留 |
| C | 是 | 清 mark bit，保留 |
| D | 是 | 清 mark bit，保留 |
| E | 否 | 回收到 freelist |
| F | 否 | 回收到 freelist |
| G | 否 | 回收到 freelist |

注意 `E/F` 虽然互相引用形成环，但从 roots 不可达，所以 mark-and-sweep 能回收它们。

优点：

- 不移动对象，对外部指针友好。
- 实现相对直接。

缺点：

- 可能产生碎片。
- sweep 要扫描堆。
- mark 可能需要显式栈，深图会占空间。

## Pointer Reversal

Mark 阶段 DFS 需要栈。Pointer reversal 的思想是临时反转对象中的指针，用堆对象自身记录回溯路径，从而减少额外栈空间。

它难点在于会临时改写对象图，所以实现和理解都较复杂。

## Reference Counting

每个对象维护引用计数：

- 新引用指向对象：计数加一。
- 引用离开对象：计数减一。
- 计数为零：对象不可达，立即回收。

优点：

- 回收分散发生，暂停时间小。
- 不需要完整遍历所有活对象。

缺点：

- 每次指针赋值都有维护开销。
- 不能回收循环垃圾。

循环例子：

```text
A -> B
B -> A
roots no longer point to A/B
```

`A`、`B` 计数都非零，但从 roots 不可达。

### 赋值序列手算

执行：

```text
p = new A
q = new B
p.f = q
q.g = p
p = nil
q = nil
```

计数表：

| 步骤 | rc(A) | rc(B) | 原因 |
|---|---:|---:|---|
| `p=new A` | 1 | 0 | `p` 指向 A |
| `q=new B` | 1 | 1 | `q` 指向 B |
| `p.f=q` | 1 | 2 | A 的字段指向 B |
| `q.g=p` | 2 | 2 | B 的字段指向 A |
| `p=nil` | 1 | 2 | root `p` 移除 |
| `q=nil` | 1 | 1 | root `q` 移除 |

最后 A/B 从 roots 不可达，但计数都为 1，reference counting 无法回收循环垃圾。

## Copying Collection

把堆分成 from-space 和 to-space。分配只在 from-space；GC 时把可达对象复制到 to-space，然后整块丢弃 from-space。

Cheney 算法用两个指针：

```text
to-space:
| copied scanned | copied unscanned | free |
                 ^scan              ^next
```

步骤：

1. roots 指向的对象复制到 to-space。
2. 旧对象留下 forwarding pointer。
3. scan 指针扫描已复制对象中的字段，发现新对象就复制。
4. scan 追上 next 时结束。

优点：

- 分配快，通常只是移动 next 指针。
- 回收后对象紧凑，减少碎片。
- 复制顺序有助于局部性。

缺点：

- 需要额外空间。
- 对象地址会变，必须更新所有指针。

### Cheney 算法手算

from-space：

```text
roots: r -> A
A fields: [B, C]
B fields: [C]
C fields: []
D fields: []    // 不可达
```

to-space 初始空。步骤：

| 步骤 | scan | next | 动作 | to-space |
|---|---|---|---|---|
| 1 | 0 | 0 | 复制 root 指向的 A，root 改指向 A' | `A'` |
| 2 | A' | after A' | 扫描 A'，发现 B/C 未复制，复制 B'、C'，A' 字段更新 | `A',B',C'` |
| 3 | B' | after C' | 扫描 B'，字段 C 已有 forwarding pointer，改指 C' | `A',B',C'` |
| 4 | C' | after C' | C' 无指针字段 | `A',B',C'` |
| 5 | scan==next | after C' | 结束，丢弃整个 from-space | `A',B',C'` |

`D` 没被复制，因此自然回收。forwarding pointer 的作用是：同一个旧对象只复制一次，并让所有指针都更新到新地址。

## Compiler Interface

GC 需要编译器支持：

| 信息 | 用途 |
|---|---|
| pointer map | 告诉 GC 哪些栈槽/寄存器是指针 |
| type descriptor | 告诉 GC 对象内部哪些字段是指针 |
| allocation protocol | 快速分配和触发 GC |
| safe point | GC 可以安全检查 roots 的程序点 |

Derived pointer 是难点：指针可能指向对象内部，例如数组元素地址。移动对象时，GC 需要知道 base pointer，才能正确更新 derived pointer。

### Pointer Map 示例

某个 safe point 的栈帧：

```text
FP+0   return address
FP-4   local i:int
FP-8   local obj:pointer
FP-12  local tmp:int
FP-16  local arr:pointer
register r1: int
register r2: pointer
```

pointer map 可写成：

```text
stack pointer slots: FP-8, FP-16
register pointer slots: r2
```

GC 只从这些位置读取 roots。若把整数 `i` 当指针，可能错误保留垃圾；若漏掉 `obj`，可能回收活对象。

type descriptor 示例：

```text
record { int x; pointer next; string name }
descriptor bitmap = 0 1 1
```

GC 扫描对象时只跟踪第 2、3 个字段。

## 弱化内容速记

### Generational GC

基于经验：多数对象很快死亡。把堆分代，频繁收年轻代，较少收老年代。需要 remembered set 和 write barrier 记录老对象指向新对象的引用。

### Incremental GC

把 GC 工作拆成小步，减少长暂停。常用 tricolor abstraction：white、gray、black。需要维护不变量，防止程序修改对象图破坏标记过程。

## 常见误区

- “不可达”是 GC 可安全判断的垃圾，不等于理论上的“以后绝不会用”。
- Reference counting 中计数非零不代表从 root 可达。
- Copying GC 必须更新所有指针，否则会悬挂到 from-space。
- GC 不是只扫描堆，还必须从 roots 开始。
- 编译器如果不给 pointer map，精确 GC 很难安全工作。
