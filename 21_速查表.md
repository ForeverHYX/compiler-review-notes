## FIRST/FOLLOW

| 集合 | 含义 |
|---|---|
| Nullable(A) | A 能推出 epsilon |
| FIRST(alpha) | alpha 能生成串的首终结符集合 |
| FOLLOW(A) | 句型中可能紧跟 A 的终结符集合 |

LL(1) 填表：

```text
for A -> alpha:
  for a in FIRST(alpha) - {epsilon}: M[A,a] = A -> alpha
  if epsilon in FIRST(alpha):
    for b in FOLLOW(A): M[A,b] = A -> alpha
```

## LR 填表

| 方法 | 归约条件 |
|---|---|
| LR(0) | 完整项在所有终结符上归约 |
| SLR(1) | 完整项 `A -> alpha .` 在 `FOLLOW(A)` 上归约 |
| LR(1) | 完整项 `[A -> alpha ., a]` 只在 `a` 上归约 |
| LALR(1) | 合并相同 core 的 LR(1) 状态 |

## 活跃变量方程

```text
out[n] = union in[s] for s in succ[n]
in[n]  = use[n] union (out[n] - def[n])
```

反向迭代直到不变。

## 寄存器分配流程

```text
Build -> Simplify -> Coalesce -> Freeze -> Spill -> Select
if actual spill:
  Rewrite Program -> rebuild
```

关键判断：

- 度数 `< K` 的节点可 simplify。
- move 源/目标无干涉时可考虑 coalesce。
- potential spill 不一定 actual spill。

| 判据/操作 | 速记 |
|---|---|
| Briggs | 合并后高阶邻居数 `< K` 可合并 |
| George | 高阶邻居要么已与目标干涉，要么是低阶 |
| actual spill rewrite | use 前 load，def 后 store，然后重做 liveness |
| caller-save | 跨 call 活跃的值放这里可能要调用前后保存 |
| callee-save | 函数入口保存、出口恢复，适合长生命周期值 |

## GC 算法对比

| 算法 | 优点 | 缺点 |
|---|---|---|
| mark-and-sweep | 不移动对象 | 碎片、扫描堆 |
| reference counting | 暂停小、及时回收 | 循环垃圾、更新开销 |
| copying collection | 分配快、紧凑 | 需要额外空间、移动对象 |
| generational GC | 利用多数对象短命 | 需要 write barrier |

## 循环优化条件

| 优化 | 需要关注 |
|---|---|
| LICM | 无副作用、支配使用、循环不变 |
| strength reduction | 归纳变量线性关系 |
| bounds-check elimination | 范围证明、支配关系 |
| loop unrolling | 余数处理、代码膨胀 |

## Dominator / Loop 速查

```text
Dom(entry) = {entry}
Dom(n) = {n} union intersection(Dom(p) for p in pred(n))
```

| 概念 | 判定 |
|---|---|
| `d dominates n` | entry 到 n 的所有路径都经过 d |
| back edge `n -> h` | `h` dominates `n` |
| natural loop | 从 tail 反向找前驱直到 header |
| preheader | 循环外进入 header 前的专门块 |
