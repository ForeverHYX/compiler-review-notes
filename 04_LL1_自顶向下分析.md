# 04 LL(1) 自顶向下分析

## 本章解决什么问题

自顶向下分析从开始符号出发，试图生成输入串。LL(1) 的核心要求是：只看一个 lookahead token，就能唯一决定用哪条产生式。

`LL(1)` 的含义：

- 第一个 `L`：从左到右扫描输入。
- 第二个 `L`：构造最左推导。
- `1`：使用 1 个向前看符号。

## 本章考试能力清单

- 概念题：能解释 LL(1)、lookahead、nullable、FIRST、FOLLOW、predictive parsing table。
- 手算题：能逐轮计算 Nullable/FIRST/FOLLOW，能构造 LL(1) 表，能用 parse stack 模拟输入串。
- 改写题：能消除直接左递归、提左公因子，并判断是否仍有 dangling else。
- 英文题：看到 `conflicting entry`、`epsilon production`、`panic-mode recovery` 能说明含义。

## 递归下降

每个非终结符写成一个函数。比如：

```text
E -> T E'
E' -> + T E' | epsilon
T -> id
```

可以写成：

```text
parseE():
  parseT()
  parseEPrime()

parseEPrime():
  if lookahead == '+':
    match('+')
    parseT()
    parseEPrime()
  else:
    return
```

如果需要回溯，效率和错误信息都可能变差；LL(1) 通过集合分析避免回溯。

## Nullable、FIRST、FOLLOW

### Nullable

`Nullable(X)` 表示 `X` 能否推出空串。

### FIRST

`FIRST(alpha)` 是 `alpha` 能推出的串的第一个终结符集合。如果 `alpha` 能推出空串，也包含 `epsilon`。

直觉：如果现在要展开 `A -> alpha`，`FIRST(alpha)` 告诉你这条产生式可能以什么 token 开头。

### FOLLOW

`FOLLOW(A)` 是在某个句型中可能紧跟在 `A` 后面的终结符集合。开始符号的 FOLLOW 通常包含 EOF。

直觉：如果 `A` 可以消失为 `epsilon`，就要看 `A` 后面可能跟什么。

## FIRST/FOLLOW 手算模板

考试里不要凭直觉一次写完，按固定迭代过程更稳。

### 计算 Nullable

```text
先把所有 Nullable(A) 设为 false
重复直到不变:
  如果有产生式 A -> epsilon，则 Nullable(A)=true
  如果有产生式 A -> X1 X2 ... Xk，且每个 Xi 都 nullable，则 Nullable(A)=true
```

### 计算 FIRST

```text
先把所有 FIRST(A) 设为空集
重复直到不变:
  对每条产生式 A -> X1 X2 ... Xk:
    从 X1 开始向右看
    如果 Xi 是终结符，把 Xi 加入 FIRST(A)，停止
    如果 Xi 是非终结符，把 FIRST(Xi)-{epsilon} 加入 FIRST(A)
    如果 Xi 不 nullable，停止
    如果所有 Xi 都 nullable，把 epsilon 加入 FIRST(A)
```

### 计算 FOLLOW

```text
FOLLOW(S) 加 EOF
重复直到不变:
  对每条产生式 A -> X1 X2 ... Xk:
    对每个非终结符 Xi:
      beta = Xi 后面的串
      把 FIRST(beta)-{epsilon} 加入 FOLLOW(Xi)
      如果 beta nullable 或 beta 为空:
        把 FOLLOW(A) 加入 FOLLOW(Xi)
```

做题时建议画三张表：`Nullable` 表、`FIRST` 表、`FOLLOW` 表。每轮只做集合增加，不要删除元素。

## LL(1) 预测分析表

对于产生式 `A -> alpha`：

1. 对每个 `a in FIRST(alpha)`，把 `A -> alpha` 填入 `M[A,a]`。
2. 如果 `epsilon in FIRST(alpha)`，对每个 `b in FOLLOW(A)`，把 `A -> alpha` 填入 `M[A,b]`。
3. 如果某个格子出现多条产生式，文法不是 LL(1)。

## LL(1) 判断清单

对同一个非终结符 `A` 的任意两条产生式：

```text
A -> alpha
A -> beta
```

必须满足：

1. `FIRST(alpha)` 和 `FIRST(beta)` 不相交。
2. 如果 `alpha` 可空，则 `FIRST(beta)` 不能和 `FOLLOW(A)` 相交。
3. 如果 `beta` 可空，则 `FIRST(alpha)` 不能和 `FOLLOW(A)` 相交。

这三条的直觉是：看一个 lookahead 时，不能同时有两条产生式可选。

## 表驱动分析过程

维护一个栈和输入：

```text
stack = [EOF, S]
while top != EOF:
  X = top(stack)
  a = lookahead(input)
  if X is terminal:
    match X with a
  else:
    use table M[X,a] to expand X
```

## 消除直接左递归

左递归会让递归下降无限递归。

原文法：

```text
A -> A alpha | beta
```

改写：

```text
A  -> beta A'
A' -> alpha A' | epsilon
```

例：

```text
E -> E + T | T
```

改为：

```text
E  -> T E'
E' -> + T E' | epsilon
```

## 提左公因子

如果多个产生式开头相同，一个 lookahead 无法决定。

原文法：

```text
A -> alpha beta1 | alpha beta2
```

改写：

```text
A  -> alpha A'
A' -> beta1 | beta2
```

## 例题：计算 FIRST/FOLLOW

文法：

```text
E  -> T E'
E' -> + T E' | epsilon
T  -> F T'
T' -> * F T' | epsilon
F  -> ( E ) | id
```

### Nullable

| 非终结符 | 是否 nullable | 原因 |
|---|---|---|
| `E` | 否 | `T` 不可空 |
| `E'` | 是 | 有 `epsilon` 产生式 |
| `T` | 否 | `F` 不可空 |
| `T'` | 是 | 有 `epsilon` 产生式 |
| `F` | 否 | 只能以 `(` 或 `id` 开始 |

### FIRST

```text
FIRST(F)  = { (, id }
FIRST(T') = { *, epsilon }
FIRST(T)  = { (, id }
FIRST(E') = { +, epsilon }
FIRST(E)  = { (, id }
```

上面旧式表达式里如果 `T -> id`，`FIRST(T)` 只有 `{id}`；这里加入括号和乘法，是为了让 FOLLOW 例题完整。

### FOLLOW

```text
FOLLOW(E)  = { EOF, ) }
FOLLOW(E') = { EOF, ) }
FOLLOW(T)  = { +, EOF, ) }
FOLLOW(T') = { +, EOF, ) }
FOLLOW(F)  = { *, +, EOF, ) }
```

关键来源：

- `FOLLOW(E)` 有 `EOF`，因为 `E` 是开始符号。
- `F -> ( E )` 让 `)` 加入 `FOLLOW(E)`。
- `E -> T E'` 中，`FIRST(E')-{epsilon}={+}` 加入 `FOLLOW(T)`；又因为 `E'` 可空，所以 `FOLLOW(E)` 也加入 `FOLLOW(T)`。
- `T -> F T'` 同理让 `*` 加入 `FOLLOW(F)`，并把 `FOLLOW(T)` 传给 `FOLLOW(F)`。

## 例题：LL(1) 表和栈模拟

继续使用上面的表达式文法。预测分析表中非空格子：

| 非终结符 | lookahead | 产生式 |
|---|---|---|
| `E` | `(`, `id` | `E -> T E'` |
| `E'` | `+` | `E' -> + T E'` |
| `E'` | `)`, `EOF` | `E' -> epsilon` |
| `T` | `(`, `id` | `T -> F T'` |
| `T'` | `*` | `T' -> * F T'` |
| `T'` | `+`, `)`, `EOF` | `T' -> epsilon` |
| `F` | `(` | `F -> ( E )` |
| `F` | `id` | `F -> id` |

模拟输入：

```text
id + id * id EOF
```

栈顶写在右边：

| 步骤 | 栈 | 输入 | 动作 |
|---|---|---|---|
| 0 | `EOF E` | `id + id * id EOF` | `E -> T E'` |
| 1 | `EOF E' T` | `id + id * id EOF` | `T -> F T'` |
| 2 | `EOF E' T' F` | `id + id * id EOF` | `F -> id` |
| 3 | `EOF E' T' id` | `id + id * id EOF` | match `id` |
| 4 | `EOF E' T'` | `+ id * id EOF` | `T' -> epsilon` |
| 5 | `EOF E'` | `+ id * id EOF` | `E' -> + T E'` |
| 6 | `EOF E' T +` | `+ id * id EOF` | match `+` |
| 7 | `EOF E' T` | `id * id EOF` | `T -> F T'` |
| 8 | `EOF E' T' F` | `id * id EOF` | `F -> id` |
| 9 | `EOF E' T' id` | `id * id EOF` | match `id` |
| 10 | `EOF E' T'` | `* id EOF` | `T' -> * F T'` |
| 11 | `EOF E' T' F *` | `* id EOF` | match `*` |
| 12 | `EOF E' T' F` | `id EOF` | `F -> id` |
| 13 | `EOF E' T' id` | `id EOF` | match `id` |
| 14 | `EOF E' T'` | `EOF` | `T' -> epsilon` |
| 15 | `EOF E'` | `EOF` | `E' -> epsilon` |
| 16 | `EOF` | `EOF` | accept |

这个表就是考试里“表驱动 LL(1) 分析”的标准写法。

## Panic-Mode Error Recovery

LL parser 查表时如果 `M[A,a]` 为空，就说明当前非终结符 `A` 无法在 lookahead `a` 下展开。简单错误恢复方法是 panic-mode：

```text
把 FOLLOW(A) 作为同步集合 sync(A)
如果 lookahead 不在 sync(A)，丢弃输入 token
如果 lookahead 在 sync(A)，弹出 A，继续分析
```

直觉：既然 `FOLLOW(A)` 是 `A` 后面合法出现的 token，那么遇到这些 token 时可以假装 `A` 这部分出错并结束，尽量恢复到外层结构。

## 常见误区

- `FIRST(A)` 看的是 A 能生成的串的开头，不是 A 后面能跟什么。
- `FOLLOW(A)` 不包含 epsilon。
- `epsilon in FIRST(alpha)` 时才需要看 FOLLOW。
- 左递归改写会改变树形结构，但不应改变语言。
- LL(1) 不等于所有文法都能改成 LL(1)。
