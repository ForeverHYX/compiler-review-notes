## ch15 函数式语言

核心问题：函数可以作为值传递，内部函数可以引用外部变量，编译器如何表示函数值？

### Closure

闭包 = 代码指针 + 环境。

```text
closure:
| code pointer |
| free var 1   |
| free var 2   |
```

如果函数引用了自由变量，这些变量必须保存在环境中。

### Closure Conversion

把函数值显式改写为 closure，并把自由变量访问改为从 environment 读取。

例：

```text
let x = 10 in
  function addx(y) = x + y
```

`addx` 的自由变量是 `x`。闭包转换后可理解为：

```text
env_addx = { x = 10 }
closure_addx = { code = addx_code, env = env_addx }

addx_code(env, y):
  x = env.x
  return x + y
```

调用 `addx(3)` 变成：

```text
CALL closure_addx.code(closure_addx.env, 3)
```

这和第 09 章 static link 类似：都是给函数补一个环境参数。区别是 closure 可以作为值逃逸，环境通常在堆上。

### Tail Call

尾调用是函数最后一步调用另一个函数。可以复用当前栈帧，避免栈增长。

## ch16 多态类型

核心问题：一个函数能否对多种类型工作，同时保持类型安全？

### Parametric Polymorphism

例：

```text
identity : forall a. a -> a
```

同一个函数对任意类型 `a` 都成立。

### Type Inference

Hindley-Milner 风格类型推断常用：

- unification
- occurs check
- generalize
- instantiate

考试若出现，重点理解“通过约束求类型”。

小例：

```text
fun f(x) = x + 1
```

生成约束：

```text
x : α
1 : int
+ 要求两边 int，所以 α = int
返回 int
```

因此：

```text
f : int -> int
```

再例：

```text
fun id(x) = x
```

没有约束要求 `x` 是具体类型，所以：

```text
id : forall α. α -> α
```

## ch17 通用数据流分析

ch10 的 liveness 是数据流分析的一个例子。ch17 泛化到其他分析：

| 分析 | 方向 | 用途 |
|---|---|---|
| reaching definitions | 前向 | 某个定义能到达哪些使用 |
| available expressions | 前向 | 公共子表达式消除 |
| live variables | 反向 | 寄存器分配 |
| copy propagation | 前向 | 替换复制变量 |

通用结构：

```text
facts_in -> transfer function -> facts_out
iterate over CFG until fixed point
```

四要素：

| 要素 | 含义 |
|---|---|
| direction | 前向或反向 |
| lattice | 数据事实的偏序结构 |
| meet operator | 合并多条路径的信息，如 union/intersection |
| transfer function | 一个节点如何改变事实 |
| boundary value | entry/exit 的初值 |

以 liveness 为例：

```text
direction = backward
meet = union
transfer = use union (out - def)
boundary at exit = empty
```

## ch19 SSA

Static Single Assignment 要求每个变量只定义一次。

```text
x1 = 1
x2 = x1 + 1
```

控制流合并处用 phi 函数：

```text
x3 = phi(x1, x2)
```

SSA 的好处：

- def-use 关系更清楚。
- 常量传播、死代码消除更容易。
- 需要 dominator 和 dominance frontier 辅助构造。

例：

```text
if cond:
  x = 1
else:
  x = 2
y = x + 3
```

SSA：

```text
if cond:
  x1 = 1
else:
  x2 = 2
x3 = phi(x1, x2)
y1 = x3 + 3
```

`phi` 表示根据控制流来自哪个前驱块选择哪个值。

## ch20 流水线与调度

核心问题：现代 CPU 可并行执行部分指令，但数据依赖和资源限制会阻止并行。

调度要考虑：

- 数据依赖图。
- 功能单元数量。
- 指令延迟。
- 分支预测。
- 软件流水。

软件流水把循环不同迭代的指令交错执行，以提高吞吐。

## ch21 存储层次

核心问题：内存访问远慢于寄存器和缓存，优化 locality 很重要。

### Cache Locality

- Temporal locality：最近访问过的数据很可能再次访问。
- Spatial locality：访问一个地址后，附近地址也可能访问。

### Loop Interchange

改变嵌套循环顺序，使内存访问更连续。

### Blocking / Tiling

把大矩阵运算分成小块，让数据在 cache 中重复使用。

行主序数组 `A[i][j]` 中，内层循环让 `j` 连续变化通常空间局部性更好：

```c
for (i=0; i<n; i++)
  for (j=0; j<n; j++)
    sum += A[i][j];   // 连续访问
```

如果写成内层 `i` 变化：

```c
for (j=0; j<n; j++)
  for (i=0; i<n; i++)
    sum += A[i][j];   // 跨行跳跃
```

cache miss 可能更多。loop interchange 的目标就是让内层循环匹配内存布局。
