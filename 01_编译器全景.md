# 01 编译器全景

## 本章解决什么问题

编译器不是一个“把 C 变成汇编”的黑盒，而是一条分阶段流水线。每个阶段只解决一类问题，并把结果交给下一阶段。理解全景之后，后面的词法、语法、类型检查、IR、寄存器分配就不会变成孤立算法。

## 从源程序到目标代码

典型编译流程：

```mermaid
flowchart LR
  S[字符流 Source] --> L[词法分析 Lexer]
  L --> T[Token 流]
  T --> P[语法分析 Parser]
  P --> A[AST]
  A --> Sem[语义分析 Semantic Analysis]
  Sem --> IR[中间表示 IR]
  IR --> Opt[机器无关优化]
  Opt --> IS[指令选择]
  IS --> RA[寄存器分配]
  RA --> Asm[目标汇编/机器码]
```

每一步都降低一种复杂度：

| 阶段 | 输入 | 输出 | 主要问题 |
|---|---|---|---|
| 词法分析 | 字符流 | Token 流 | 哪些字符组成一个词 |
| 语法分析 | Token 流 | 语法树/AST | Token 能否组成合法结构 |
| 语义分析 | AST | 带类型/绑定信息的 AST | 名字有没有声明，类型是否匹配 |
| IR 生成 | AST | 中间表示 | 把语言结构降到更接近机器的形式 |
| 优化 | IR | 更好的 IR | 消除冗余、改善性能 |
| 指令选择 | IR | 抽象汇编 | 用哪些机器指令实现 IR |
| 寄存器分配 | 无限临时变量 | 有限寄存器 | 哪些值能共用寄存器 |

## 编译器、解释器、JIT、AOT

狭义编译器在运行前生成目标程序。解释器直接执行源程序或字节码。JIT 在程序运行时把热点代码编译为机器码。AOT 在运行前把中间表示或字节码编译到本地机器码。

关键区别不是“快不快”，而是翻译发生在什么时候、输出能否独立运行。

## 前端、中端、后端

- 前端关注源语言：词法、语法、语义、AST。
- 中端关注机器无关 IR：数据流分析、循环优化、SSA 等。
- 后端关注目标机器：指令选择、寄存器分配、调用约定、指令调度。

把编译器分成这些层次的好处是模块化。假设有 `N` 种源语言和 `M` 种目标机器，直接互相翻译要 `N*M` 个编译器；共享 IR 后大约是 `N+M` 个前后端。

## Tiger 编译器主线

教材用 Tiger 语言作为贯穿案例。Tiger 编译器的意义不是让你学 Tiger，而是用一个小语言把完整编译器阶段串起来：

```text
Tiger source
  -> Tokens
  -> AST
  -> Semantic checked AST
  -> Tree IR
  -> Canonical IR
  -> Basic blocks / traces
  -> Assembly instructions
  -> Flow graph
  -> Liveness
  -> Interference graph
  -> Colored registers
  -> Final assembly
```

## 例题

给程序片段：

```c
position = initial + rate * 60;
```

可能的阶段输出：

- 词法分析：`ID(position) ASSIGN ID(initial) PLUS ID(rate) TIMES NUM(60) SEMI`
- 语法分析：赋值语句，右侧是加法表达式，加法右侧含乘法表达式。
- 语义分析：检查 `position`、`initial`、`rate` 是否声明，`rate * 60` 是否类型正确。
- IR 生成：可能生成三地址码：

```text
t1 = rate * 60
t2 = initial + t1
position = t2
```

## 常见误区

- 语法正确不代表语义正确：`x + true` 可能能构成表达式，但类型不合法。
- AST 不是 Parse Tree：AST 删除很多只为解析服务的中间节点。
- IR 不是汇编：IR 通常机器无关，而汇编依赖目标架构。
- 优化不只发生在后端：中端优化通常基于 IR，前端也可能做简单常量折叠。

2. 给出表达式 `a = b + c * d` 的 Token 流和一份三地址码。

## 术语中英对照

| English | 中文 | 考试提示 |
|---|---|---|
| compiler | 编译器 | 翻译并保持语义等价 |
| interpreter | 解释器 | 直接执行源程序或中间形式 |
| just-in-time compiler, JIT | 即时编译器 | 运行时编译 |
| ahead-of-time compiler, AOT | 预先编译器 | 运行前编译 |
| cross compiler | 交叉编译器 | 在一种平台上生成另一平台代码 |
| lexical analysis | 词法分析 | 字符流到 Token 流 |
| parsing, syntax analysis | 语法分析 | Token 流到语法结构 |
| semantic analysis | 语义分析 | 名称解析和类型检查 |
| intermediate representation, IR | 中间表示 | 前后端之间的桥梁 |
| instruction selection | 指令选择 | IR 到目标指令 |
| register allocation | 寄存器分配 | 临时变量映射到物理寄存器 |
| runtime system | 运行时系统 | 支持调用、内存、GC 等 |
