# 23 练习参考答案

本文件给出各章练习的参考答案。编译原理很多题不是唯一答案，尤其是 CFG、AST、IR、trace、寄存器分配，只要满足语义和算法约束即可。答案重点写“怎么判断”和“关键步骤”。

## 01 编译器全景

1. 错误分类：
   - `int 3abc;`：词法错误或词法/语法边界错误。若语言规定标识符不能以数字开头，`3abc` 无法作为合法标识符。
   - `if (x > 0 y = 1;`：语法错误，括号不匹配，条件表达式没有正确结束。
   - `int x; x = "hello";`：语义错误，语法结构合法，但把字符串赋给 `int` 变量类型不匹配。
2. `a = b + c * d` 的阶段关注点：
   - 词法分析：识别名字 `a,b,c,d`、赋值号、加号、乘号。
   - 语法分析：识别为赋值语句，右侧乘法优先于加法。
   - 语义分析：检查变量声明和类型。
   - IR/代码生成：把右侧拆成先乘后加再赋值。
3. IR 降低成本：没有 IR 时，`N` 种源语言到 `M` 种机器要 `N*M` 套翻译；有共享 IR 后，只需 `N` 个前端和 `M` 个后端。

## 02 词法分析

1. 正则表达式示例：
   - 标识符：`[A-Za-z_][A-Za-z0-9_]*`
   - 十进制整数：`[0-9]+`
   - 简单浮点数：`[0-9]+ "." [0-9]+`
   - 单行注释：`"//" [^\n]*`
2. `(a|b)*abb` 的自动机检查点：
   - Thompson NFA 应包含三类结构：`a|b` 的分支、`*` 的回边和跳过边、末尾连续 `a b b`。
   - 子集构造时 DFA 状态是 NFA 状态集合。
   - 终态是包含 NFA 最终状态的 DFA 集合。
   - 可接受串示例：`abb`、`aabb`、`babb`、`abababb`；不可接受串：`ab`、`abba`。
3. `elsewhere else = ==`：
   - `elsewhere` 是 `ID`，因为最长匹配比关键字 `else` 长。
   - `else` 是 `ELSE`，因为长度相同时关键字规则在 ID 规则前。
   - `=` 是 `ASSIGN`。
   - `==` 是 `EQ`，因为最长匹配优先于单个 `=`。
4. DFA 最小化步骤答案应包含：
   - 初始划分：终态组与非终态组。
   - 对每个输入字符检查组内状态是否跳到相同组。
   - 若不同则拆分。
   - 稳定后的每组是最小 DFA 的一个状态。

## 03 CFG、推导、二义性

1. `id + id * id` 在二义文法 `E -> E+E | E*E | id` 下：
   - 一棵树以 `*` 为根：`(id + id) * id`。
   - 一棵树以 `+` 为根：`id + (id * id)`。
2. 括号匹配 CFG：
   - `S -> S S | ( S ) | epsilon`
   - 或只生成单个嵌套结构：`S -> ( S ) S | epsilon`
3. `E -> E - E | id` 二义。`id - id - id` 可解释为 `(id-id)-id` 或 `id-(id-id)`。
4. 左结合且 `*` 高于 `-`：
   ```text
   E -> E - T | T
   T -> T * F | F
   F -> id | ( E )
   ```

## 04 LL(1)

1. 文法：
   ```text
   S -> A B
   A -> a A | epsilon
   B -> b B | c
   ```
   Nullable：
   - `A` nullable。
   - `B` 不 nullable。
   - `S` 不 nullable。
   FIRST：
   - `FIRST(A) = {a, epsilon}`
   - `FIRST(B) = {b, c}`
   - `FIRST(S) = {a, b, c}`
   FOLLOW：
   - `FOLLOW(S) = {EOF}`
   - `FOLLOW(A) = FIRST(B) = {b, c}`
   - `FOLLOW(B) = FOLLOW(S) = {EOF}`
2. LL(1) 表：
   - `M[S,a] = S -> A B`
   - `M[S,b] = S -> A B`
   - `M[S,c] = S -> A B`
   - `M[A,a] = A -> a A`
   - `M[A,b] = A -> epsilon`
   - `M[A,c] = A -> epsilon`
   - `M[B,b] = B -> b B`
   - `M[B,c] = B -> c`
3. `E -> E - T | T` 消除直接左递归：
   ```text
   E  -> T E'
   E' -> - T E' | epsilon
   ```
4. `if` 文法提左公因子：
   ```text
   S  -> if E then S S' | other
   S' -> else S | epsilon
   ```
   dangling else 没有靠提左公因子彻底消除；还需要语言规则规定 `else` 归最近未匹配的 `if`，或改写成 matched/unmatched 文法。

## 05 LR(0) 与 SLR

1. 构造项集时必须先增广：
   ```text
   S' -> S EOF
   S  -> E
   E  -> E + T | T
   T  -> id
   ```
   `I0` 至少包含：
   ```text
   S' -> . S EOF
   S  -> . E
   E  -> . E + T
   E  -> . T
   T  -> . id
   ```
2. 表构造规则：
   - 终结符边填 `shift`。
   - 非终结符边填 `GOTO`。
   - 完整项 `A -> alpha .` 在 LR(0) 中对所有终结符填 reduce；SLR 中只对 `FOLLOW(A)` 填 reduce。
3. 分析 `id + id EOF` 的动作序列应类似：
   ```text
   shift id
   reduce T -> id
   reduce E -> T
   shift +
   shift id
   reduce T -> id
   reduce E -> E + T
   reduce S -> E
   accept
   ```
   具体状态号取决于你的项集编号。
4. 冲突判断：若某状态既有 `A -> alpha .` 又有 `B -> beta . a gamma`，同一 lookahead 上可能 shift/reduce。SLR 只有当 shift 的终结符不在 `FOLLOW(A)` 中时才能消除。

## 06 LR(1)、LALR、Yacc

1. LR(1) closure：
   对 `[A -> alpha . B beta, a]` 和每个 `B -> gamma`，加入 `[B -> . gamma, b]`，其中 `b in FIRST(beta a)`。
2. 归约位置：
   - LR(0)：所有终结符。
   - SLR：`FOLLOW(A)`。
   - LR(1)：item 自带 lookahead。
3. LALR 合并示例：
   ```text
   [A -> x ., a] 和 [A -> x ., b]
   ```
   合并为 core 相同的状态，lookahead 为 `{a,b}`。
4. Yacc 语义动作示例：
   ```yacc
   exp : exp '+' exp { $$ = A_OpExp(PLUS, $1, $3); }
       | exp '*' exp { $$ = A_OpExp(TIMES, $1, $3); }
       | NUM         { $$ = A_IntExp($1); }
       ;
   ```

## 07 AST

1. `a[i] = b + 1` 的 AST 可为：
   ```text
   Assign
   ├── SubscriptVar(a, i)
   └── Op(+, Var(b), Int(1))
   ```
2. 节点字段示例：
   - `IfExp(test, then, else)`
   - `WhileExp(test, body)`
   - `CallExp(funcName, args)`
3. 语义动作：
   ```yacc
   exp : exp '+' exp { $$ = A_OpExp(PLUS, $1, $3); }
       | ID          { $$ = A_VarExp(A_SimpleVar($1)); }
       | NUM         { $$ = A_IntExp($1); }
       ;
   ```
4. 位置信息用于报错，例如“第 12 行 undefined variable x”。没有位置信息，语义分析只能说“某处错误”，不利于调试。

## 08 语义分析

1. 嵌套 `let` 环境变化要体现 shadowing：
   - 进入作用域：压 marker。
   - 插入同名变量：新 binding 覆盖旧 binding。
   - 退出作用域：弹到 marker，恢复旧 binding。
2. record 类型是否等价取决于语言规则。Tiger 中通常按 name equivalence：即使结构相同，两个不同类型名也不等价。
3. 
   ```text
   type a = b
   type b = c
   type c = a
   ```
   非法，因为只有 name type 之间形成环，没有经过 record 或 array。
4. 函数 two-pass：
   - 第一遍把所有函数名、参数类型、返回类型加入 `VEnv`。
   - 第二遍在这个扩展环境中检查函数体。
5. `f(x+1, "s")`：
   - 查 `f` 的函数类型。
   - 检查参数数量。
   - 检查 `x+1` 类型是否为第 1 个形参类型。
   - 检查 `"s"` 是否为第 2 个形参类型。
   - 表达式类型为 `f` 的返回类型。

## 09 活动记录与 Static Link

1. 三层嵌套时，最内层 frame 的 static link 指向词法外层 frame，外层再指向更外层 frame。访问最外层变量要沿 static link 走两次。
2. 变量 escape：
   - 被内层函数引用：escape。
   - 取地址或引用传递：escape。
   - 只在当前函数内计算：通常不 escape。
3. caller-save：调用前由调用者保存调用后还要用的值。callee-save：被调用者若使用该寄存器，必须保存并恢复。
4. dynamic link 表示调用者，不表示词法外层。非局部变量按词法作用域查找，所以要 static link。

## 10 IR Tree

1. `x := a + b * c`：
   ```text
   MOVE(access(x),
        BINOP(PLUS, access(a),
             BINOP(MUL, access(b), access(c))))
   ```
2. `if a < b then x := 1 else x := 2`：
   ```text
   CJUMP(<, a, b, Ltrue, Lfalse)
   LABEL Ltrue
   MOVE(x, CONST 1)
   JUMP Ldone
   LABEL Lfalse
   MOVE(x, CONST 2)
   LABEL Ldone
   ```
3. `while i < n do i := i + 1`：
   ```text
   LABEL Ltest
   CJUMP(<, i, n, Lbody, Ldone)
   LABEL Lbody
   MOVE(i, BINOP(PLUS, i, CONST 1))
   JUMP Ltest
   LABEL Ldone
   ```
4. `Ex` 产生值，`Nx` 只产生副作用，`Cx` 产生控制流。`Cx -> Ex` 可用临时变量在 true/false 分支赋 `1/0`。

## 11 规范化、基本块与 Trace

1. 含 `ESEQ(s,e)` 的表达式应把 `s` 提前成语句，再使用 `e`。例如 `BINOP(+, ESEQ(s,e1), e2)` 可改为先执行 `s`，再算 `BINOP(+, e1, e2)`，前提是顺序安全。
2. basic block：
   - 以 `LABEL` 开头。
   - 中间无 `LABEL`。
   - 以 `JUMP/CJUMP` 结尾。
3. trace 是按可能执行顺序串联 basic blocks。合法 trace 不改变控制流，只调整块顺序和必要 jump。
4. `CJUMP` 三种处理：
   - 下一块是 false label：直接保留。
   - 下一块是 true label：反转条件。
   - 下一块都不是：插入新的 false label 和 jump。

## 12 指令选择

1. 合法 tiling 要覆盖每个 IR 节点且 tile 不重叠。
2. Maximal Munch 从根开始选最大匹配 tile，再递归处理叶子子树。
3. 动态规划：每个节点记录最低成本 tile 和总成本，后序计算。
4. `dst/src/jumps`：
   - `dst` 是指令定义的临时变量。
   - `src` 是指令读取的临时变量。
   - `jumps` 是跳转目标 label 集合。

## 13 活跃变量分析

1. 先逐行列 `use/def`。
2. 用方程：
   ```text
   out[n] = union in[s] for successors s
   in[n] = use[n] union (out[n] - def[n])
   ```
3. 对线性代码从后往前迭代通常一轮即可；有循环时要多轮直到不变。
4. 干涉图：对每条定义 `d`，把 `d` 与 `live-out` 中仍活跃的变量连边。move 指令可特殊处理，记录 coalescing 候选。

## 14 寄存器分配

1. `K=3` 图着色：
   - 删除度数 `<3` 的节点并压栈。
   - 没有低度节点时选 potential spill。
   - 弹栈时为每个节点选择未被邻居使用的颜色。
2. Briggs 判据：合并后高阶邻居数量少于 `K` 时可安全合并。
3. actual spill 重写：
   - 每次使用前插入 load 到新 temporary。
   - 每次定义后插入 store。
   - 重新做 liveness 和寄存器分配。
4. caller-save 寄存器在 call 后可能被破坏；若值跨 call 活跃，分配到 caller-save 会带来保存恢复成本。

## 15 完整编译器串联

1. 小程序阶段输出：字符流 -> token -> AST -> 类型检查结果 -> IR -> 规范 IR -> 抽象汇编 -> CFG/liveness -> 寄存器分配后汇编。
2. bug 定位：
   - token 错：lexer。
   - 括号/优先级错：parser。
   - 未声明变量/类型错：semantic。
   - 外层变量错：frame/static link/translate。
   - 寄存器覆盖：liveness/regalloc/calling convention。
3. `Frame` 同时服务前端变量访问和后端机器布局，因此不能完全归入某一端。
4. procedure fragment 保存函数代码；string fragment 保存字符串常量。

## 16 垃圾回收

1. mark-and-sweep：从 roots 可达的对象保留，不可达对象回收。
2. reference counting：每次指针赋值维护计数，计数为 0 回收。
3. 循环垃圾示例：`A -> B` 且 `B -> A`，roots 不指向它们，计数仍非零。
4. Cheney copying：roots 指向对象复制到 to-space，旧对象留 forwarding pointer，scan 指针扫描已复制对象字段。
5. pointer map 告诉 GC 栈槽/寄存器哪些是指针；type descriptor 告诉 GC 对象内部哪些字段是指针。

## 17 面向对象语言

1. `B extends A` 对象布局通常是：
   ```text
   descriptor
   A fields
   B fields
   ```
2. `obj.f()` 动态派发：
   ```text
   d = obj.descriptor
   fptr = d.vtable[slot(f)]
   call fptr(obj, args)
   ```
3. static dispatch 编译期确定目标；dynamic dispatch 运行时按对象实际类查 vtable。
4. 子类 override 方法时保持同一 vtable slot，只替换函数指针。
5. 多继承困难：多个父类字段布局和方法槽位可能冲突，父类视角的 offset 不再天然一致。

## 18 循环优化

1. Dominator 迭代：
   ```text
   Dom(entry)={entry}
   Dom(n)=AllNodes for others
   Dom(n)={n} union intersection(Dom(p) for p in pred(n))
   ```
2. back edge `n -> h` 且 `h` 支配 `n`。natural loop 从 `{h,n}` 开始，反向加入 `n` 的前驱直到稳定。
3. loop-invariant：操作数都在循环外定义，或由已知循环不变式定义，并且移动安全。
4. strength reduction：把 `4*i` 这类每轮乘法改为新变量 `t` 每轮加 4。
5. bounds check 消除需要证明循环内所有下标范围满足数组边界。
6. unrolling by 2 要生成主循环处理成对迭代，再用尾循环处理余数。

## 19 教材后半拓展速读

1. 自由变量：函数体中使用但不是函数参数或局部声明的变量。closure record 保存 code pointer 和自由变量环境。
2. 类型推断：从表达式生成类型约束，用 unification 解约束。
3. SSA：每次赋值产生新版本；控制流合流处插入 `phi`。
4. locality：连续访问数组通常空间局部性更好；重复访问近期数据时间局部性更好。
5. blocking：把大矩阵分块，让小块留在 cache 中多次使用，减少 cache miss。

## 20 综合练习题

### 题 1

`ifx == if else2 = 123`：

```text
ID(ifx) EQ IF ID(else2) ASSIGN NUM(123)
```

### 题 2

可接受答案应包含 Thompson 的并、闭包、连接结构。DFA 终态是包含 NFA 最终状态的子集。可用测试串检验：接受 `abb`、`aabb`、`babb`；拒绝 `ab`、`abba`。

### 题 3

按终态/非终态初分，再对每个输入拆分。最终同组状态必须对所有输入跳到同组。

### 题 4

同第 04 章答案：

```text
Nullable(A)=true, Nullable(B)=false, Nullable(S)=false
FIRST(A)={a,epsilon}
FIRST(B)={b,c}
FIRST(S)={a,b,c}
FOLLOW(S)={EOF}
FOLLOW(A)={b,c}
FOLLOW(B)={EOF}
```

### 题 5

```text
E  -> T E'
E' -> + T E' | epsilon
T  -> F T'
T' -> * F T' | epsilon
F  -> id | ( E )
```

### 题 6

核心答案同第 05 章。状态号可不同，但必须从增广文法和 `I0` closure 开始，ACTION/GOTO 与 DFA 一致。

### 题 7

LR(0) 中完整项会在所有终结符上归约，可能和 `*` 的 shift 冲突。SLR 只在 `FOLLOW(E)` 或 `FOLLOW(T)` 上归约。LR(1) 用 item 自带 lookahead 更精确。

### 题 8

```text
Assign
├── SubscriptVar(a, Var(i))
└── Op(+)
    ├── Var(b)
    └── Op(*, Var(c), Int(2))
```

### 题 9

`in x` 绑定到内层 `var x:string`。退出 let 后恢复外层 `x:int`。

### 题 10

`var a : A := nil` 合法。`var b : B := a` 在 name equivalence 下非法，因为 `A` 与 `B` 是不同 record 类型名。

### 题 11

同第 10 章 while 模板。

### 题 12

```text
MOVE(TEMP a, CONST 1)
BINOP(PLUS, TEMP a, TEMP b)
```

### 题 13

`L3` 后若直接接 `L4` 可视为 fall-through；若基本块规则要求每块以跳转结尾，则补 `JUMP L4`。trace 可为 `L1 -> L3 -> L4` 和 `L2`，或根据 CJUMP 反转安排。

### 题 14

支持 offset addressing 时一条大 tile 覆盖 `MEM(PLUS(fp,-4))`，生成 `load r, -4(fp)`。不支持时需要先加地址再 load，至少两条指令。

### 题 15

同第 13 章例题：

```text
1 in={} out={a}
2 in={a} out={a,b}
3 in={a,b} out={c}
4 in={c} out={}
```

干涉边至少有 `a-b`。

### 题 16

一种 3 色着色：

```text
a=R1
b=R2
c=R3
d=R1
```

因为 `d` 不与 `a` 相邻。

### 题 17

`a` 和 `b` 无干涉且保守判据允许时可 coalesce。不能无条件合并，因为合并会增加度数，可能破坏 K-colorable。

### 题 18

最内层访问最外层变量：沿 static link 从 inner frame 到 middle frame，再到 outer frame，然后按变量 offset 访问。

### 题 19

`A/B` 从 roots 可达，保留。`C/D` 不可达，mark-and-sweep 回收。reference counting 无法回收 `C/D` 的环。copying GC 会把 `A/B` 复制到 to-space 并更新 roots。

### 题 20

`B` 对象：

```text
descriptor
a
b
```

`A.vtable[slot f]=A.f`；`B.vtable[slot f]=B.f`，新增 `slot g=B.g`。`x.f()` 通过 descriptor 找 vtable，再取 `slot f` 调用。

### 题 21

`a+b` 是 loop-invariant。`i` 是基本归纳变量，`x=4*i` 是派生归纳变量。强度削弱：维护 `x`，每轮 `x=x+4`。bounds check 消除需要证明 `0 <= x < length(arr)` 在所有迭代成立。
